Tunable properties are cluster properties that are specifically designed for tuning a cluster.

For information on how to edit tunable properties, refer to [Configuring Cluster Properties](../../cluster-administration/cluster-property-configuration).

## Properties

### abort_index_segment_size

Capacity of an abort index segment.

**Units**: number of transactions

**Default**: 50000

**Range**: [1, ...] 

---

### abort_timed_out_transactions_interval_ms

Timer timeout to check for and abort inactive transactions.

**Units**: milliseconds

**Default**: 10000

**Range**: [1, ...] 

---

### alter_topic_cfg_timeout_ms

Time to wait for entries replication in controller log when executing alter configuration request.

**Units**: milliseconds

**Default**: 5000

**Range**: [1, ...] 

---

### append_chunk_size

Size of direct write operations to disk.

**Units**: bytes

**Default**: 16 KB

**Range**: [4096, 32 MB]. Must be 4096-byte aligned. 

---

### cloud_storage_cache_check_interval

Timeout to check if cache eviction should be triggered.

**Units**: milliseconds

**Default**: 30000

**Range**: [1, ...]

---

### cloud_storage_enable_remote_read

Enable remote read for all topics.

**Type**: boolean

**Default**: false

---

### cloud_storage_enable_remote_write

Enable remote write for all topics.

**Type**: boolean

**Default**: false

---

### cloud_storage_initial_backoff_ms

Initial backoff time for exponetial backoff algorithm.

**Units**: milliseconds

**Default**: 100

**Range**: [1, ...]

---

### cloud_storage_manifest_upload_timeout_ms

Manifest upload timeout.

**Units**: milliseconds

**Default**: 10000

**Range**: [1, ...]

---

### cloud_storage_max_connection_idle_time_ms

Maximum HTTPS connection idle time.

**Units**: milliseconds

**Default**: 5000

**Range**: [1, ...]

---

### cloud_storage_reconciliation_interval_ms

Interval at which the archival service runs reconciliation.

**Units**: milliseconds

**Default**: 1000

**Range**: [1, ...]

---

### cloud_storage_segment_max_upload_interval_sec

Duration that a segment can be kept in local storage without uploading it to remote storage.

**Units**: seconds

**Default**: null (infinite duration)

**Range**: [1, ...] or null (infinite)

---

### cloud_storage_segment_upload_timeout_ms

Log segment upload timeout.

**Units**: milliseconds

**Default**: 30000

**Range**: [1, ...]

---

### cloud_storage_upload_ctrl_d_coeff

Derivative coefficient of the upload proportional-integral-derivative (PID) controller. For details, see [Upload controller](../../data-management/tiered-storage#upload-controller).

**Type**: double

**Default**: 0.0

**Range**: [0.0, ...]

---

### cloud_storage_upload_ctrl_max_shares

Maximum number of IO and CPU shares that the archival upload can use. For details, see [Upload controller](../../data-management/tiered-storage#upload-controller).

**Units**: number of IO and CPU shares

**Default**: 1000

**Range**: [1, ...]

---

### cloud_storage_upload_ctrl_min_shares 

Minimum number of IO and CPU shares that the archival upload can use. For details, see [Upload controller](../../data-management/tiered-storage#upload-controller).

**Units**: number of IO and CPU shares

**Default**: 100

**Range**: [1, ...]

---

### cloud_storage_upload_ctrl_p_coeff

Proportional coefficient of the upload proportional-integral-derivative (PID) controller. For details, see [Upload controller](../../data-management/tiered-storage#upload-controller).

**Type**: double

**Default**: -2.0

**Range**: (VERIFY)

---

### cloud_storage_upload_ctrl_update_interval_ms

Sampling interval of the upload proportional-integral-derivative (PID) controller. For details, see [Upload controller](../../data-management/tiered-storage#upload-controller).

**Units**: milliseconds

**Default**: 60000

**Range**: [1, ...]

---

### compacted_log_segment_size

Size of a compacted log segment.

**Type**: int

**Units**: bytes

**Default**: 256 MB

**Range**: [1 MB, ...]

---

### compaction_ctrl_backlog_size

Size of the compaction backlog of the backlog controller. If empty (null), defaults to 10% of disk capacity.

**Type**: int

**Units**: bytes

**Default**: null

**Range**: [1 MB, ...]

---

### compaction_ctrl_d_coeff

Derivative coefficient for compaction proportional-integral-derivative (PID) controller.

**Type**: double

**Default**: 0.2

**Range**: [0.0, ...]

---

### compaction_ctrl_i_coeff

Integral coefficient for compaction proportional-integral-derivative (PID) controller.

**Type**: double

**Default**: 0.0

**Range**: [0.0, ...]

---

### compaction_ctrl_max_shares

Maximum number of IO and CPU shares that the compaction process can use. 

**Units**: int

**Default**: 1000

**Range**:

---

### compaction_ctrl_min_shares

Minimum number of IO and CPU shares that the compaction process can use. 

**Type**: int

**Units**: number of shares

**Default**: 10

**Range**:

---

### compaction_ctrl_p_coeff

Proportional coefficient for compaction proportional-integral-derivative (PID) controller. Must be negative to decrease the compaction backlog when compaction shares increase.

**Type**: double

**Default**: -12.5

**Range**: [..., 0.0)

---

### compaction_ctrl_update_interval_ms

Sampling interval of the compaction proportional-integral-derivative (PID) controller.

**Units**: milliseconds

**Default**: 30000

**Range**: [1, ...]

---

### controller_backend_housekeeping_interval_ms

Duration between iterations of the controller backend's housekeeping loop.

**Units**: milliseconds

**Default**: 1000

**Range**: [1, ...]

---

### coproc_max_batch_size

Maximum size of one topic read in coprocessing mode. For details, see [Enable data transforms](../../labs/data-transform/#enable-data-transforms).

**Units**: bytes

**Default**: 32 KB

**Range**: [1, ...]

---

### coproc_max_inflight_bytes

Maximum size of inflight data when sending to the Wasm engine. For details, see [Enable data transforms](../../labs/data-transform/#enable-data-transforms).

**Units**: bytes

**Default**: 10485760 (10 MB)

**Range**: [1, ...]

---

### coproc_max_ingest_bytes

Maximum size of input logs in memory. For details, see [Enable data transforms](../../labs/data-transform/#enable-data-transforms).

**Units**: bytes

**Default**: 655360 (64 KB)

**Range**: [1, ...]

---

### coproc_offset_flush_interval_ms

Interval for which coprocessor offsets are flushed to disk. For details, see [Enable data transforms](../../labs/data-transform/#enable-data-transforms).

**Units**: milliseconds

**Default**: 300000

**Range**: [1, ...]

---
